# Oracle hand-rolled-arithmetic allowlist (credibility ratchet)
# =============================================================================
# Frozen baseline for scripts/check_oracle_handrolled.sh.
#
# Each name is a function in oracle/driver.ml that performs binary64
# arithmetic WITHOUT routing through a Coq-extracted function in
# oracle/extracted.ml.  Entries fall into TWO categories:
#
#   (1) TRANSITIONAL -- pre-dated the ratchet; technical debt to be migrated
#       to extracted definitions (docs/oracle-handroll-migration.md).  This
#       sub-set may only SHRINK.  (Currently empty: all migrated.)
#
#   (2) INTERFACE-BOUNDARY -- a SANCTIONED, PERMANENT exception (owner policy,
#       2026-06-03).  A mode that must emit the same primitive value the
#       Java (JTS) / C# (NTS) implementation computes for differential
#       testing, where that value is TRANSCENDENTAL (atan2 / sqrt / acos /
#       trig) and therefore has NO Coq-extractable form (Flocq exposes no
#       Batan2 etc.).  Such a mode legitimately hand-rolls float: there is no
#       extractable reference to route through, and the float IS the interface.
#       Each entry must name its mode + the Java/C# call it mirrors, and where
#       possible derive from an EXACT companion mode (so it rounds only once
#       past certified rational algebra).
#
# RULES (enforced by CI -- the script compares detected vs. allowed):
#   - A new TRANSITIONAL kernel fails review (that sub-set only shrinks).
#   - A new INTERFACE-BOUNDARY kernel is allowed ONLY with the justification
#     comment above its line (mode + Java/C# call + exact companion).
#   - When any kernel is migrated/removed, delete its line in the same change,
#     or CI fails on the stale entry.
#
# One function name per line; `#` comments and blank lines ignored.
# =============================================================================

# === INTERFACE-BOUNDARY (sanctioned, permanent) ============================
#
# --- ARC_LENGTH (issue #64) -------------------------------------------------
# Mode ARC_LENGTH emits the literal float arc length s = r * Theta, the value
# the JTS/NTS curve implementation computes (Java Math.sqrt + Math.acos).
# Arc length is transcendental, so it has NO Coq-extractable form (Flocq has
# no Batan2/Bacos); the float IS the differential-test interface.  The kernel
# applies sqrt/acos to the EXACT rational invariants of arc_invariants_q (the
# certifiable ARC_LENGTH_INVARIANTS_EXACT mode: r2, cos Theta0, major-arc flag), so it
# rounds only ONCE past certified algebra (theories/ArcLength.chord_subtended_sq,
# theories/AngleBetween.cos_angle_between).
run_arc_length

# --- ARC_AREA (M-AREA-CP, issue #64) ----------------------------------------
# Mode ARC_AREA emits the float circular-SEGMENT area of one arc,
# A_seg = (r^2/2)(Theta - sin Theta) -- transcendental (Theta), no
# Coq-extractable form; the float is the JTS/NTS curve-area interface value.
# Refactored to mirror ARC_LENGTH: it applies acos + sin to the EXACT rational
# invariants of arc_invariants_q (the certifiable ARC_AREA_INVARIANTS_EXACT
# mode: r2, cos Theta0, sin^2 Theta0, major-arc flag), so it rounds only ONCE
# past certified algebra.  (Supersedes main 53d61c7's hand-rolled shoelace stub,
# which had bypassed the then-broken ratchet.)
run_arc_area

# --- arc_bisector_unit (shared helper for the centroid modes) ---------------
# Unit angular bisector of arc A->B->C about the centre, toward the arc midpoint.
# Float helper (+,-,*,/,sqrt only, no transcendental) shared by ARC_CENTROID and
# ARC_AREA_CENTROID to place the centroid offset along the true angular bisector
# m = sgn*(uA+uC)/|uA+uC| -- NOT along the mid control point B, which is an
# arbitrary (non-midpoint) point on the arc for general CircularStrings.  Part of
# the interface-boundary centroid-POINT computation (the float IS the interface);
# off the EXACT circumcentre_q centre.
arc_bisector_unit

# --- ARC_CENTROID (C-LIN, issue #64/#69) ------------------------------------
# Mode ARC_CENTROID emits the centroid POINT of one circular arc,
# O + (2*r*sin(Theta/2)/Theta) * m  with m = arc_bisector_unit -- the offset
# 2*r*sin(Theta/2)/Theta is transcendental (asin), no Coq-extractable form; the
# float is the JTS/NTS CircularString.getCentroid() interface value (weighted
# per-arc by ARC_LENGTH).  Applies asin to the EXACT arc_invariants_q invariants +
# the exact circumcentre_q, rounding only ONCE past certified algebra.  Spec
# proven in theories/ArcCentroid.v (offset = 2r*sin(Theta/2)/Theta; semicircle ->
# 2r/PI; full turn -> 0; 0 <= offset <= r).
run_arc_centroid

# --- ARC_AREA_CENTROID (C-AREA, issue #64/#69) ------------------------------
# Mode ARC_AREA_CENTROID emits the centroid POINT of one circular SEGMENT,
# O + (4*r*sin^3(Theta/2)/(3*(Theta - sin Theta))) * m  with m = arc_bisector_unit
# -- transcendental (asin/sin), no Coq-extractable form; the JTS/NTS curve
# area-centroid interface value.  Mirrors ARC_AREA (same Theta - sin Theta Taylor),
# off the EXACT arc_invariants_q / circumcentre_q kernel.  Spec proven in
# theories/ArcAreaCentroid.v (semicircle -> 4r/(3*PI); full turn -> 0).
run_arc_area_centroid

# --- ARC_DISTANCE (D-PT, issue #64/#69) -------------------------------------
# Mode ARC_DISTANCE emits the shortest distance from a point to one circular arc.
# Radial foot distance ||P-O| - r| when the foot lies on the arc, else
# min(|P-A|,|P-C|); the on-arc-sector membership uses atan2 (CCW angle interval)
# -- transcendental (atan2/sqrt), no Coq-extractable form -- off the EXACT
# circumcentre_q centre.  The JTS/NTS point-to-arc distance interface value.
run_arc_distance

# --- point_on_arc_sector (shared helper for arc-arc / distance sector tests) -
# Is a circle point within an arc's angular span? CCW angle-interval test via
# atan2 (transcendental) -- shared by ARC_ARC_XY (and matching ARC_DISTANCE's
# inline sector test).  Interface-boundary helper off the exact circumcentre.
point_on_arc_sector

# --- ARC_ARC_XY (N-AA, issue #64 #5b) ---------------------------------------
# Mode ARC_ARC_XY emits the intersection POINTS of two circular arcs.  Two
# circumcircles meet on the radical line; the coordinates carry an irrational
# sqrt(discriminant) (no Coq-extractable form, like ARC_LINE_XY), computed off
# the EXACT circumcentre_q centres/radii so only the final sqrt/atan2 rounds.
# Points kept iff in both arc spans (point_on_arc_sector).  The JTS/NTS arc-arc
# intersection interface value (N-AA); the coordinate soundness proof is the
# deferred #5b frontier (theories/ArcArcSound.v has the slice).
run_arc_arc_xy

# --- ARC_SEGMENT_XY (N-AL, issue #224 W1) -----------------------------------
# Mode ARC_SEGMENT_XY emits the intersection POINTS of a circular arc with a
# line SEGMENT (the segment analogue of ARC_ARC_XY; replaces the single-point
# ARC_LINE_XY).  The coordinates carry an irrational sqrt(r^2 - d^2) (no
# Coq-extractable form, like ARC_LINE_XY / ARC_ARC_XY); the count decision
# r^2 - d^2 >/=/< 0 and the perpendicular foot are computed in EXACT Q off the
# circumcentre_q centre/radius, so only the final sqrt and emitted coordinates
# round.  Points kept iff in the arc span (point_on_arc_sector) AND on the
# segment (0 <= t <= 1).  The JTS/NTS CircularArcs.intersectSegment interface
# value (N-AL); coordinate existence proven in theories/ArcSegmentCircles.v
# (line_circle_radical_point / arc_line_circle_intersect).
run_arc_segment_xy

# --- ARC_ARC_DISTANCE (D-AA, JTS #1195 §7) ----------------------------------
# Mode ARC_ARC_DISTANCE emits the shortest distance between two circular arcs.
# Min over: interior radial feet on the line O1 O2 when the circumcircles are
# disjoint (gap d-r1-r2 external / |r1-r2|-d internal); 0 when they intersect in
# both sweeps; and the four endpoint-vs-other-arc point-to-arc distances (the
# ARC_DISTANCE kernel).  Transcendental (sqrt + atan2 sector membership), no
# Coq-extractable form, off the EXACT circumcentre_q centres/radii so only the
# final sqrt/atan2 round.  The JTS/NTS arc-arc distance interface value; the
# disjoint circle-to-circle core is proven in theories/ArcArcDistance.v
# (two_circles_dist_radial / arc_arc_dist_external), sweep clamp deferred.
run_arc_arc_distance

# --- ARC_SEGMENT_DISTANCE (D-SL, JTS #1195 §7) ------------------------------
# Mode ARC_SEGMENT_DISTANCE emits the shortest distance between a circular arc
# and a line SEGMENT.  Min over: the radial point of the arc over the
# perpendicular foot G of O on the segment line (gap perp - r) when G is on the
# segment, the line is outside the circle, and the radial point is in the sweep;
# 0 when the line meets the circle and a crossing is on the segment and in the
# sweep; and the endpoints (segment ends to arc via the ARC_DISTANCE kernel, arc
# chord-ends to the segment).  Transcendental (sqrt + atan2 sector membership),
# no Coq-extractable form, off the EXACT circumcentre_q centre + exact-Q foot so
# only the final sqrt/atan2 round.  The JTS/NTS arc-segment distance interface
# value; the line-outside-circle core is proven in theories/ArcSegmentDistance.v
# (circle_line_dist_radial / arc_segment_dist_external), sweep/segment clamp deferred.
run_arc_segment_distance

# --- RING_SIMPLE (V-CS / V-CP, JTS #1195 §7) --------------------------------
# Mode RING_SIMPLE decides whether a closed curve ring (chords / arcs) is simple
# (no two non-adjacent segments meet; consecutive segments meet only at their
# connecting vertex).  Composes the arc-arc / arc-segment / chord-chord
# intersection primitives over every segment pair: centres/radii are EXACT
# (circumcentre_q) but the in-sweep membership uses the atan2 point_on_arc_sector
# test (transcendental, no Coq-extractable form), so the float IS the interface
# (same boundary as ARC_ARC_XY / point_on_arc_sector).  The JTS/NTS
# isSimple/isValid interface value (V-CS/V-CP); the NOT_SIMPLE-witness soundness
# is proven in theories/CurveRingSimple.v (curve_ring_not_simple_of_witness),
# completeness + reflex-arc spans deferred.
run_ring_simple

# --- ARC_OFFSET_XY (OFF / BUF-1 / BUF-NEG, JTS #1195 §7) ---------------------
# Mode ARC_OFFSET_XY emits the offset (buffer boundary) of a circular arc at
# signed distance d: the concentric arc of radius r+d, each control point mapped
# radially to O + ((r+d)/r)(P-O).  The scale (r+d)/r carries r = sqrt(r2) (the
# circumradius is irrational), so the emitted coordinates round once past the
# EXACT circumcentre_q centre/radius^2 -- the float IS the JTS/NTS buffer-offset
# interface value.  Backed by theories/ArcOffsetThreePoint.v
# (arc_offset_preserves_arc: offset of a valid arc is a valid arc, same centre,
# radius r+d; radial_offset_dist_exact: controls at distance |d|) and the EMPTY
# boundary (r+d <= 0) witness ArcOffset.inner_offset_past_center_not_at_distance.
run_arc_offset_xy

# --- POINT_IN_CURVE_RING (V-CP / CP_VALID holes-inside-shell, JTS #1195 §7) ---
# Mode POINT_IN_CURVE_RING decides whether a query point is inside the TRUE
# curved region of a curve ring, by ARC-AWARE ray casting: the rightward ray
# crosses each chord by the usual edge test and each ARC where the horizontal
# line meets the arc's circle (x = ox +- sqrt(r^2 - (py-oy)^2)) inside the arc's
# sweep.  Centres/radii are EXACT (circumcentre_q) but the arc-line crossing
# needs sqrt and the sweep test needs atan2 (point_on_arc_sector) -- no
# Coq-extractable form, the float IS the JTS/NTS point-in-curve-polygon
# interface (same boundary as ARC_ARC_XY / point_on_arc_sector).  The sound
# conservative floor (inscribed chord-polygon containment) is proven in
# theories/CurvePolygonValid.v; the true-region (Jordan) soundness is the
# deferred frontier, pinned by oracle/gen_point_in_curve_ring_tests.py.
run_point_in_curve_ring

# --- RING_ORIENTATION (V-CP / CP_VALID sector orientation, JTS #1195 §7) ------
# Mode RING_ORIENTATION emits the TRUE signed area of a curve ring (chord
# shoelace + each arc's signed circular-segment area sb*r^2*(theta - sin theta))
# -> CCW/CW.  Centres/radii^2 are EXACT (arc_invariants_q) and the sweep sign sb
# is exact-Q, but the swept angle theta needs acos/sin -> transcendental, no
# Coq-extractable form, the float IS the JTS/NTS curve-polygon-area/orientation
# interface (same boundary as ARC_AREA).  The orientation ALGEBRA (segment-area
# reversal, classifier, shell/hole convention) is proven in
# theories/CurvePolygonOrientation.v; the topological "sign = inside orientation"
# (Jordan) is deferred, pinned by oracle/gen_ring_orientation_tests.py.
run_ring_orientation

# --- HOLES_DISJOINT (V-CP / CP_VALID holes-mutually-disjoint, JTS #1195 §7) ---
# Mode HOLES_DISJOINT decides whether two curve-polygon hole rings' regions are
# disjoint: NOT_DISJOINT iff their boundaries meet (arc-arc / arc-segment /
# chord-chord intersection, the RING_SIMPLE pair primitive) or one is nested in
# the other (the POINT_IN_CURVE_RING arc-aware ray cast).  Both components carry
# sqrt + atan2 (sweep / circle-line intersection) -> no Coq-extractable form, the
# float IS the interface (same boundary as ARC_ARC_XY / point_on_arc_sector).
# The structural soundness (a boundary meeting or nesting witness => not
# disjoint) is proven in theories/CurvePolygonDisjoint.v; true-region
# containment soundness is the deferred frontier, pinned by
# oracle/gen_holes_disjoint_tests.py.
run_holes_disjoint

# --- CURVE_RELATE_MATRIX (R-PR, JTS #1195 §7) --------------------------------
# Mode CURVE_RELATE_MATRIX COMPUTES the full 9-cell DE-9IM intersection matrix of
# two curve geometries (TRUE OGC convention), generalizing HOLES_DISJOINT.  It is
# a COMPOSITION of the already-allowlisted kernels (run_holes_disjoint's arc-arc /
# arc-segment / chord-chord boundary-intersection primitives + the
# POINT_IN_CURVE_RING arc-aware ray cast), plus an arc-by-angle boundary sampler
# (atan2/sin/cos) and a bounding-box grid for the open cells.  All carry
# sqrt + atan2 -> no Coq-extractable form, the float IS the interface (same
# boundary as ARC_ARC_XY / point_on_arc_sector).  The reused primitives are inner
# `let`s of run_curve_relate_matrix, so the detector attributes their arithmetic
# to this enclosing function -- one allowlist entry suffices.  The point-set
# DE-9IM spec + algebraic laws (well-formedness, exteriors-meet => EE nonempty,
# transpose-under-swap, interior/boundary-meet "disjoint" characterization,
# curated OGC witness matrices) are proven in theories/RelateCurveMatrix.v; the
# computed cell-DIMENSION soundness (Jordan/overlay) addressed in S13
# (RelateCurveMatrix + RelateNG); vectors continue to pin numeric behaviour.
run_curve_relate_matrix

# --- BUFFER_REGION (BUF-1 / BUF-NEG, JTS #1195 §7) ---------------------------
# Mode BUFFER_REGION ASSEMBLES the offset boundary of a closed curve ring at
# signed distance d and emits it + its TRUE signed area -- the buffer-region
# certificate.  It composes the run_arc_offset_xy radial homothety (arc offset to
# radius r+sigma*d, sigma = arc sweep-sign * ring orientation = convex/concave
# outward radial sign), the BufferOffset chord normal offset, CurveRoundJoin
# round-join arcs at convex corners, and the run_ring_orientation signed-area
# kernel (chord shoelace + arc sector areas).  All carry sqrt + atan2/asin off the
# EXACT circumcentre_q / arc_invariants_q, so the emitted coordinates and area
# round once past certified Q -- the float IS the JTS/NTS buffer interface.  The
# reused primitives are inner `let`s, so the detector attributes their arithmetic
# to this enclosing function.  Backed (3-axiom) by the offset-assembly rungs
# (CurveRingOffset .. CurveOffsetAssemblyTotal.curve_ring_offset_total_valid,
# CurveRoundJoin.round_join_arc_valid) + theories/CurveBufferArea.v (boundary
# validity + area algebra: d=0 identity, EMPTY/safety decision, orientation
# preservation); the geometric "signed area = true Minkowski buffer area" is the
# deferred P2 frontier, pinned by oracle/gen_buffer_region_tests.py.
run_buffer_region
