# Oracle hand-rolled-arithmetic allowlist (credibility ratchet)
# =============================================================================
# Frozen baseline for scripts/check_oracle_handrolled.sh.
#
# Each name is a function in oracle/driver.ml that performs binary64
# arithmetic WITHOUT routing through a Coq-extracted function in
# oracle/extracted.ml.  Entries fall into TWO categories:
#
#   (1) TRANSITIONAL -- pre-dated the ratchet; technical debt to be migrated
#       to extracted definitions (docs/oracle-handroll-migration.md).  This
#       sub-set may only SHRINK.  (Currently empty: all migrated.)
#
#   (2) INTERFACE-BOUNDARY -- a SANCTIONED, PERMANENT exception (owner policy,
#       2026-06-03).  A mode that must emit the same primitive value the
#       Java (JTS) / C# (NTS) implementation computes for differential
#       testing, where that value is TRANSCENDENTAL (atan2 / sqrt / acos /
#       trig) and therefore has NO Coq-extractable form (Flocq exposes no
#       Batan2 etc.).  Such a mode legitimately hand-rolls float: there is no
#       extractable reference to route through, and the float IS the interface.
#       Each entry must name its mode + the Java/C# call it mirrors, and where
#       possible derive from an EXACT companion mode (so it rounds only once
#       past certified rational algebra).
#
# RULES (enforced by CI -- the script compares detected vs. allowed):
#   - A new TRANSITIONAL kernel fails review (that sub-set only shrinks).
#   - A new INTERFACE-BOUNDARY kernel is allowed ONLY with the justification
#     comment above its line (mode + Java/C# call + exact companion).
#   - When any kernel is migrated/removed, delete its line in the same change,
#     or CI fails on the stale entry.
#
# One function name per line; `#` comments and blank lines ignored.
# =============================================================================
